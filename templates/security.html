Security

Our security practices and responsible disclosure policy

Security Architecture

  • Input Sanitization — All URLs are validated and sanitized before processing to prevent injection attacks.
  • Request Timeouts — All external requests have strict timeouts (5-10 seconds) to prevent resource exhaustion.
  • Rate Limiting — API endpoints are rate-limited to prevent abuse (60 requests/minute per IP).
  • HTTPS Everywhere — All external API communications use TLS 1.2 or higher.
  • Memory-Only Processing — Scan data never touches persistent storage.

Dependency Management

We regularly audit dependencies for known vulnerabilities:

  • Flask — Web framework (latest stable)
  • Requests — HTTP library with timeout enforcement
  • BeautifulSoup4 — HTML parsing
  • dnspython — DNS resolution
  • python-whois — WHOIS lookups

Responsible Disclosure

We take security seriously and appreciate the work of security researchers. If you believe you've found a vulnerability:

  • Email: help@demarickwebb.dev
  • Allow reasonable time for remediation before public disclosure
  • Provide detailed reproduction steps

We commit to:

  • Acknowledging receipt within 48 hours
  • Providing a timeline for fixes
  • Not taking legal action against good-faith research
