Security

Last updated: {{ last_updated or '2025-10-19' }}

Platform Posture

  • Rate limiting applied to scanning endpoints to deter abuse.
  • Input validation and output encoding across the stack.
  • Timeouts for outbound requests and safe user agent string.

Dependencies

We pin core dependencies in requirements.txt and periodically review security advisories. We recommend running scans inside an isolated environment.

External Integrations

Optional services (Google Safe Browsing, URLHaus, OTX) are accessed via HTTPS. API keys, when used, should be provided via environment variables and not committed to source control.

Responsible Disclosure

If you discover a vulnerability, please email demarick54@gmail.com with details. We appreciate responsible disclosure and will aim to respond as quick as possible.
